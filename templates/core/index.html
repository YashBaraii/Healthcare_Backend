🩺 Healthcare Backend API

Built with Django, DRF, PostgreSQL, and JWT Authentication

🎯 Objective

Build a secure backend for managing users, patients, doctors, and assignments using RESTful APIs, JWT auth, and PostgreSQL.

🧰 Technologies Used

  • Django & Django REST Framework
  • PostgreSQL
  • JWT Authentication (SimpleJWT)
  • Python Decouple for environment configs
  • CORS Headers

🔐 Authentication APIs

  • POST /api/auth/register/ - Register a new user
  • POST /api/auth/login/ - Login and receive JWT tokens

👤 Patient APIs

  • POST /api/patients/ - Create patient
  • GET /api/patients/ - List user's patients
  • GET /api/patients/<id>/ - View patient
  • PUT /api/patients/<id>/ - Update patient
  • DELETE /api/patients/<id>/ - Delete patient

👨‍⚕️ Doctor APIs

  • POST /api/doctors/ - Create doctor
  • GET /api/doctors/ - List all doctors
  • GET /api/doctors/<id>/ - View doctor
  • PUT /api/doctors/<id>/ - Update doctor
  • DELETE /api/doctors/<id>/ - Delete doctor

🔗 Patient-Doctor Mapping APIs

  • POST /api/mappings/ - Assign doctor to patient
  • GET /api/mappings/ - View all mappings
  • GET /api/mappings/<patient_id>/ - View assigned doctors
  • DELETE /api/mappings/delete/<id>/ - Remove mapping

📦 Data Models

🔐 User

  • email – Primary unique identifier (used for login)
  • username – Required by Django
  • password – Encrypted password

👤 Patient

  • name – Full name (string)
  • age – Age (integer)
  • gender – 'M', 'F', or 'O'
  • address – Text address
  • created_by – ForeignKey to User

👨‍⚕️ Doctor

  • name – Full name
  • specialization – Field of medicine
  • experience – Years of experience (integer)
  • contact – Contact email or number

🔗 PatientDoctorMapping

  • patient – ForeignKey to Patient
  • doctor – ForeignKey to Doctor
  • 🔁 Each mapping is unique per patient-doctor pair

🔒 Security & Validation

  • Only patient creators can edit/delete their records
  • Validations for mapping (duplicate prevention, existence check)
  • All secrets in .env
  • Global error handling with friendly messages

🧪 Testing & Features

  • All APIs tested in Postman with JWT
  • CORS enabled for future frontend integration
  • Pagination enabled for large data sets

🚀 Deployment Ready

  • Environment-ready via Python-Decouple
  • Database: PostgreSQL
